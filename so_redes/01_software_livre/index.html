O Universo do Software Livre e Open Source

Explorando a revolução tecnológica baseada na colaboração global e na liberdade do código.

FILOSOFIA

Software Livre vs. Open Source

Software Livre é um movimento social focado na ética e nas liberdades do usuário. Já o Open Source é uma metodologia de desenvolvimento pragmática, focada na eficiência e segurança através da colaboração.

Exemplo: O Software Livre prioriza o direito de modificar o código por princípio moral, enquanto o Open Source utiliza o código aberto para criar produtos mais robustos e competitivos no mercado.

As 4 Liberdades do Software Livre

Definições fundamentais estabelecidas pela Free Software Foundation (FSF).

Executar o programa para qualquer propósito.

Estudar como o programa funciona e adaptá-lo às suas necessidades (exige acesso ao código-fonte).

Redistribuir cópias para ajudar o próximo.

Melhorar o programa e distribuir essas melhorias para que toda a comunidade se beneficie.

O Projeto GNU

Criado por Richard Stallman em 1983, o nome é um acrônimo recursivo para 'GNU's Not Unix'. O projeto visava criar um sistema operacional completo e livre, desenvolvendo ferramentas essenciais como GCC, Emacs e Bash.

Evolução do Ecossistema

Marcos históricos que levaram à criação do sistema operacional moderno.

1983

Fundação do GNU

Richard Stallman inicia o projeto para criar um sistema operacional livre compatível com Unix.

1991

Kernel Linux

Linus Torvalds publica o kernel Linux, preenchendo a lacuna de núcleo do projeto GNU.

União

GNU/Linux

A integração das ferramentas GNU com o kernel Linux forma o sistema operacional completo que utilizamos hoje.

TECNOLOGIA

Kernel

O kernel é o 'cérebro' ou a camada mais baixa de um sistema operacional, atuando como a ponte direta entre o hardware e os softwares.

Exemplo: Ele gerencia o processador, a memória RAM e os drivers de dispositivos, garantindo que os aplicativos funcionem sem corromper o sistema.

Funções Principais do Kernel

O kernel é responsável pela orquestração dos recursos do computador.

  • Gerenciamento de recursos (processador)
  • Gerenciamento de memória RAM
  • Controle de drivers de hardware
  • Segurança e isolamento de processos

Perfis de Distribuições Linux

Por que existem tantas distros? Cada uma atende a um perfil de necessidade específico.

Iniciante

Foco em facilidade de uso e configuração pronta (ex: Ubuntu, Mint).

Estabilidade

Foco em confiabilidade para sistemas críticos (ex: Debian).

Inovação

Integração rápida de novas tecnologias (ex: Fedora).

Personalização

Controle total para usuários avançados (ex: Arch Linux).

Licenças de Software

Diferenças fundamentais entre as abordagens de licenciamento.

Copyleft (GPL)

Garante que o software permaneça livre para sempre.

  • Obrigatório abrir código
  • Herança de liberdade
  • Foco social

Permissivas (MIT/BSD)

Flexibilidade para uso em produtos proprietários.

  • Uso comercial livre
  • Menos restrições
  • Foco corporativo
