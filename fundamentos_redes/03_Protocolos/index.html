Fundamentos de Redes de Computadores

Explorando os protocolos que sustentam a comunicação global e a arquitetura da internet.

FUNDAMENTOS

Protocolo de Rede

Um protocolo de rede é uma 'língua' comum que permite que dispositivos diferentes, como computadores e smartphones, se comuniquem de forma padronizada, independentemente do fabricante ou sistema operacional.

Exemplo: O uso de um navegador web para acessar um site, onde o protocolo HTTP garante que o servidor e o cliente entendam a requisição.

Modelo TCP/IP

A arquitetura prática que sustenta a internet, organizada em quatro camadas funcionais.

1

Aplicação

Interface direta com o usuário (HTTP, SMTP, FTP).

2

Transporte

Entrega fim a fim de dados (TCP e UDP).

3

Rede

Roteamento de pacotes via endereços lógicos (IP, ICMP).

4

Enlace

Comunicação física e endereçamento MAC.

TCP vs UDP

Diferenças fundamentais na filosofia de transporte de dados.

TCP

Orientado à conexão, prioriza a confiabilidade.

  • Three-way handshake
  • Garante entrega
  • Ordem correta
  • Uso: Web, E-mail

UDP

Sem conexão, prioriza a velocidade.

  • Envio direto
  • Baixa latência
  • Sem verificação
  • Uso: Streaming, VoIP

Características das Classes IP

Como o endereçamento é distribuído conforme o tamanho da rede.

  • Classe A: Redes extremamente grandes (1 octeto para rede).
  • Classe B: Redes de médio a grande porte (2 octetos para rede).
  • Classe C: Redes pequenas, comuns em empresas e residências (3 octetos para rede).
  • Classe D e E: Reservadas para multicast e fins experimentais.

IPv4 vs IPv6

A evolução necessária para a escalabilidade da internet.

IPv4

Padrão original de 32 bits.

  • 4,3 bilhões de endereços
  • Uso de NAT
  • Esgotamento iminente

IPv6

Novo padrão de 128 bits.

  • 340 undecilhões de endereços
  • Segurança IPsec nativa
  • Autoconfiguração

Protocolos de Suporte

Protocolos auxiliares essenciais para diagnóstico e mapeamento.

  • ICMP: Diagnóstico e mensagens de erro (ex: ping).
  • ARP: Mapeamento de IP para endereço físico (MAC).
  • RARP: Descoberta de IP a partir do MAC (obsoleto).

Como o computador descobre o endereço físico (MAC) de um destino?

O protocolo ARP atua como um tradutor entre o mundo lógico do IP e o mundo físico da rede local.

Funcionamento do ARP

O processo de resolução de endereços em uma rede local.

  • O computador conhece o IP de destino.
  • Ele envia uma requisição broadcast na rede perguntando quem possui aquele IP.
  • O dispositivo com o IP correspondente responde com seu endereço MAC.
  • O computador armazena essa relação em sua tabela ARP.
DIAGNÓSTICO

ICMP

Protocolo de controle que gerencia mensagens de erro e diagnóstico na camada de rede.

Exemplo: O comando 'ping', que utiliza ICMP para verificar se um host está ativo e medir o tempo de resposta.

LEGADO

RARP

Protocolo utilizado por máquinas sem disco para descobrir seu próprio endereço IP a partir do endereço MAC.

Exemplo: Estações de trabalho antigas que precisavam de um servidor para inicializar sua configuração de rede.

APLICAÇÃO

DNS

Sistema de nomes de domínio que atua como a 'lista telefônica da internet', traduzindo nomes legíveis em endereços IP.

Exemplo: Ao digitar 'google.com', o DNS traduz esse nome para o endereço IP do servidor do Google.

Dicas para Profissionais

Boas práticas para manter redes robustas e eficientes.

  • Sempre monitore a latência para decidir entre TCP e UDP.
  • Mantenha o DNS configurado corretamente para evitar falhas de acesso.
  • Planeje a transição para IPv6 para garantir a longevidade da infraestrutura.
  • Entenda o fluxo ARP para solucionar problemas de conectividade local.
