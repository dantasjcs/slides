FUNDAMENTOS

Criptografia

Derivado do grego 'kryptós' (oculto) e 'gráphein' (escrever), é a ciência de transformar informações legíveis em dados ininteligíveis para garantir a segurança.

Exemplo: Transformar dados sensíveis em texto cifrado (ciphertext) para que, mesmo interceptados, permaneçam inacessíveis sem a chave correta.

Pilares da Segurança da Informação

A criptografia é a ferramenta primordial para sustentar os pilares da tríade CIA e garantir a confiança digital.

  • Confidencialidade: Acesso restrito apenas a partes autorizadas.
  • Integridade: Garantia de que a informação não foi manipulada.
  • Autenticidade: Prova da origem real da mensagem.
  • Não Repúdio: Impossibilidade do emissor negar o envio da mensagem.

A criptografia garante a disponibilidade dos dados?

Não. Embora proteja o conteúdo, a criptografia não impede ataques de negação de serviço (DDoS) ou a perda da chave de descriptografia, que tornam os dados indisponíveis.

Evolução Histórica

A história da criptografia é um ciclo de inovação e superação pela criptoanálise.

1

Cifra de César

Método clássico de substituição por deslocamento, hoje trivialmente quebrável.

2

Complexidade eletromecânica superada por falhas operacionais e computação primitiva.

3

DES para AES

Transição de padrões de 56 bits para algoritmos robustos de 256 bits contra força bruta.

Lições de Alan Turing

A quebra da Enigma demonstrou que a segurança de um sistema não depende apenas do algoritmo, mas também da disciplina operacional e da ausência de hábitos previsíveis.

Criptografia Simétrica vs. Assimétrica

Modelos distintos de operação para diferentes necessidades de segurança.

Simétrica

Usa a mesma chave para cifrar e decifrar.

  • Alta velocidade
  • Ideal para grandes volumes
  • Desafio na distribuição de chaves

Assimétrica

Usa um par de chaves (pública e privada).

  • Resolve distribuição de chaves
  • Baseada em problemas matemáticos
  • Processamento mais lento

Melhores Práticas de Segurança

Diretrizes essenciais para o desenvolvimento web seguro.

Uso de funções como Argon2 ou bcrypt para armazenar senhas de forma unidirecional.

Cabeçalho que força o navegador a utilizar apenas conexões HTTPS.

Evitar hardcoding de chaves; utilizar cofres como AWS Secrets Manager ou HashiCorp Vault.

Implementação de Hash de Senha

Exemplo conceitual de armazenamento seguro utilizando funções de hash com salt.

JavaScript 
const bcrypt = require('bcrypt');
const saltRounds = 12;
const password = 'user_password';

// Hash seguro
const hash = await bcrypt.hash(password, saltRounds);
