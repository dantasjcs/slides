Fundamentos da Segurança da Informação

O papel do técnico na proteção contra ameaças digitais e falhas humanas.

Objetivos da Segurança (Tríade CID)

Os pilares fundamentais que todo profissional de TI deve proteger:

  • Confidencialidade: Garantir que a informação seja acessada apenas por pessoas autorizadas.
  • Integridade: Assegurar que os dados não sejam alterados indevidamente.
  • Disponibilidade: Garantir que os sistemas estejam acessíveis quando necessário.
AMEAÇAS

Malware

Abreviação de 'malicious software', engloba qualquer código ou programa desenvolvido para se infiltrar em sistemas, causar danos, roubar dados ou obter acesso não autorizado.

Exemplo: Um script malicioso embutido em um anexo de e-mail que criptografa arquivos do usuário.

Tipos de Malware

Principais ameaças que exploram vulnerabilidades técnicas:

Vírus

Código que se anexa a arquivos legítimos e depende de execução humana.

Worms

Programas autônomos que se replicam via rede sem precisar de hospedeiro.

Trojans

Disfarçam-se de software legítimo para abrir portas dos fundos.

Spywares

Coletam dados de navegação e teclas digitadas silenciosamente.

Adwares

Exibem anúncios intrusivos e podem redirecionar para sites maliciosos.

Ciclo de Vida de um Vírus

Como o vírus se comporta dentro de um sistema infectado:

1

Hospedagem

O código malicioso se anexa a um arquivo executável ou documento legítimo.

2

Ativação

O usuário executa o arquivo hospedeiro, disparando o código do vírus.

3

Replicação

O vírus infecta outros arquivos do sistema, espalhando a ameaça.

Vírus vs. Worms

A diferença técnica fundamental: Vírus exigem um arquivo hospedeiro e interação humana para se propagar, enquanto Worms são autônomos e exploram vulnerabilidades de rede para se espalhar rapidamente.

Cavalos de Troia

A ilusão do software legítimo.

O Trojan não se replica sozinho. Ele se disfarça de utilitário, jogo ou atualização para enganar o usuário. Uma vez executado, ele abre portas para acesso remoto e roubo de dados.

Como o Spyware monitora o usuário sem ser detectado?

Ele opera silenciosamente em segundo plano, capturando hábitos, senhas e telas, enviando tudo para um servidor remoto.

Variações de Spyware

Ferramentas de espionagem digital:

Registram cada tecla digitada pelo usuário, sendo eficazes para roubar senhas e dados bancários.

Capturam imagens da tela em intervalos ou cliques, permitindo ver exatamente o que o usuário faz.

Exibem anúncios intrusivos e frequentemente coletam dados de navegação para fins publicitários.

Evolução da Engenharia Social

A transição do foco técnico para o foco humano.

Passado

Hacking de Sistemas

Foco total em explorar falhas de software e protocolos de rede.

Presente

Hacking Humano

Exploração da falibilidade humana através de manipulação e persuasão.

Gatilhos Psicológicos

Como atacantes manipulam a vítima:

Urgência

Criação de pressão temporal.

  • Bloqueio de conta
  • Ação imediata

Autoridade

Fingir ser suporte ou chefia.

  • Confiança cega
  • Medo de punição

Medo

Ameaça de consequências.

  • Multas
  • Processos

O Processo do Phishing

Etapas da 'pescaria digital' para captura de dados:

  • Criação de página falsa idêntica a um serviço legítimo.
  • Envio de mensagem (e-mail/SMS) com link malicioso.
  • Indução do usuário a inserir credenciais.
  • Captura dos dados em tempo real pelo atacante.
PHISHING

Variações de Phishing

O Phishing possui níveis de sofisticação: o Spear Phishing é direcionado com dados da vítima, enquanto o Whaling foca exclusivamente em executivos de alto nível.

Exemplo: Um e-mail personalizado citando o nome do seu gerente para solicitar uma transferência bancária.

Técnicas de Manipulação

Métodos comuns de engenharia social:

Pretexting

Criação de um cenário falso para obter informações.

Baiting

Uso de iscas físicas ou digitais para infectar sistemas.

Tailgating

Seguir alguém para acessar áreas restritas fisicamente.

90%+

Ataques iniciados por erro humano

Constante

Evolução das ameaças

Defesas Técnicas (Hardening)

Medidas de proteção na infraestrutura:

  • Atualizações constantes de sistemas e softwares.
  • Uso de EDR para análise comportamental de ameaças.
  • Implementação de MFA (Autenticação de Múltiplos Fatores).

"A segurança da informação não é um estado final, mas um processo contínuo de vigilância e adaptação."

Defesas Humanas

Construindo uma cultura de segurança:

  • Treinamento e conscientização contínua dos usuários.
  • Políticas claras de uso de dispositivos e senhas.
  • Simulações de ataques para identificar vulnerabilidades.
CULTURA

Cultura de Segurança

A integração da segurança como pilar fundamental da TI, onde todos os usuários compreendem seu papel na proteção da infraestrutura.

Exemplo: Realizar verificações de identidade antes de fornecer acesso a dados sensíveis.
